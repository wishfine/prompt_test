# 物理难度近期实验结果与标签口径

更新时间：2026-07-20

## 1. 评测对象与统一口径

近期实验统一使用以下 1066 道题：

- 题目文件：`data/labeled/physics_difficulty_tiku_data_0714_1000.jsonl`
- 实际题目数：1066
- 模型：`doubao-seed-2.0-lite`
- Lite 服务端温度：固定为 1
- 五档顺序：送分题、基础题、中等题、拔高题、压轴题

本文所有指标均从当前工作区结果文件重新计算，不引用历史对话中的四舍五入数字。`accuracy` 表示五档完全一致率；`within-one` 表示预测与参考标签相差不超过一档的比例；`MAE` 是五档序号的平均绝对误差；严重偏差表示相差两档及以上。

1. 同一预测结果对 GPT 初裁与 GPT 复核主标签的准确率通常相差约 0.8—1.5 个百分点，说明标签复核确实改变了一批边界题。
2. 教师标签与两套 GPT 标签存在明显系统口径差异，因此约 59% 的教师一致率不能直接解释为模型能力下降。
3. accuracyfix 三次对 GPT 复核主标签平均达到 73.45%，是当前严格准确率最高且有三次稳定性数据的版本。
4. final-candidate 增加规则后三次平均降至 71.86%，说明继续堆规则会产生负收益。
5. erroraudit 与 erroraudit-guard 单次结果基本持平；后者严格准确率少 1 题，但可接受等级命中更多。

## 6. 冻结决定

首轮评级 Prompt 和当前 `gpt56_hybrid` 后处理在本阶段冻结。后续实验不得通过继续修改首轮 Prompt 来追逐 1066 题准确率，而是新增独立、可关闭、可审计的二阶段校验 Pipeline：

1. 冻结版 Lite 产生首轮 raw、features、reasoning 和最终等级；
2. 确定性风险路由只选择结构冲突或边界风险题；
3. `doubao-seed-2.0-mini` 在 `temperature=0` 下进行盲审，不接收首轮等级或标签；
4. 只有高置信度、证据完整、与确定性风险方向一致的相邻档结论才自动写回；
5. 两档以上分歧、低置信度或证据冲突只记录，不自动跨档；
6. GPT/教师标签只用于运行结束后的离线评估。

Pipeline 是否进入生产，以 1066 题上的净增正确数、改对/改错比、严重偏差、路由覆盖率和 Token 成本共同决定。
